# R2TW: Unit Viewer


> Total War: Rome II のユニット一覧ビューアー

## はじめに

----

* 公式のエンサイクロペディアがビューアーとして不便すぎる
* [Royal Military Academy](http://www.honga.net/totalwar/rome2/)は素晴らしいが、それでも思ったように一覧比較しづらい
* のでビューアーを書いた

## 使い方

----

* ファクション別ユニット表示
 * 上メニューの`勢力別ユニット`＞右メニューの`ファクション一覧`からファクションを選択
 * ファクションの一覧は軍隊グループ毎にまとめている
 * 軍隊グループ毎に使用可能なユニットが決められているので、同じ行のファクションは基本的に同じユニットになる
 * そこにファクション固有のユニットやファクション固有の傭兵がいくつか加わる
* 全てのユニット表示
 * 上メニューの`全ユニット`を選択
 * たくさん出てくるので見づらい
* ユニットリスト
 * 項目ソートとカラム幅を任意に変更できる
 * デフォルトでは 兵科＞兵種＞雇用費 の順でソートしている。リセットボタンで戻せる
 * shift押しながら項目名をクリックすることで複数項目でのソートができる
 * グラフ表示幅の数値を変えることでグラフ表示されている項目の幅を一律変更できる
* その他説明
 * プレイアブルなファクションは大きめに表示している
 * 装備品アイコンは`audio_material`というプロパティでなんとなく分類して表示
 * 装備品アイコンにマウスを載せると詳細プロパティをtooltip表示
 * 海上ユニットと攻城兵器は今のところ表示していない（理由：陸上ユニットほど大差無い、リストが見づらくなる、単純に面倒）

## ユニット性能・戦闘メカニクスについて

----

```
※ 性能表をまとめるにあたりフォーラムなどを調べた覚え書きのようなものなので、正確では無いかもしれません。
```

* 攻撃力・防御力
 * 戦闘の優劣を決める基本の値。ダメージではなく戦闘スキル
 * この数値の優劣によりどれだけ攻撃をヒットさせられるかという確率判定に使われる
 * 防御力はユニットの基本防御力と盾の防御力が合算されたものになる
* 白兵武器・射撃武器ダメージ
 * 上記でヒット判定されてから実際に与えるダメージの基本量
 * `APダメージ`(Armor Piercing:相手の装甲無視の貫通攻撃)と`通常ダメージ`の2種類がそれぞれ設定されており、ゲーム上では合算して表示されている。
 * 本ビューアのグラフでは左の濃い部分をAPダメージ、右の薄い部分を通常ダメージとして表示している
 * さらに相手のユニット（歩兵/騎馬/象）に対する`vsボーナス`設定があり、本ビューアではその中の最高値をグラフに足して表示している（アイコンにマウスを載せると詳細が見られる）。グラフでは最大ボーナスが加算された表示になっているが、数値自体はAP+基本ダメージで表示。（※対ユニットボーナスはダメージじゃなくて攻撃力に加算されるものかも？）
 * APダメージとは別に真偽値の`armor_piercing`属性もある。昔のTWでは`armor_piercing`属性だけだったようだが、rome2ではどういう扱いなのか？
 * APダメージ高い武器としてはトラキア兵のRhomphaiaやダキア兵のFalxなど。両手持ちなので代わりに盾装甲が皆無
* 装甲値
 * 発生したダメージを減衰させる値
 * `鎧`と`盾`の装甲値を合算したもの
 * 本ビューアのグラフではグラフでは左の濃い部分を鎧の装甲値、右の薄い部分を盾の装甲値として表示している
* 盾
 * 正面と左側面からの攻撃に対してのみ適用され、背後から攻撃された場合は無効化される
 * 防御力、装甲値それぞれの値を持ち、前述の条件を満たすことで加算される
 * さらに投射物をブロックする確率を持つ
* 突撃ボーナス
 * ステータスが突撃中になっているときだけ追加で発生するボーナス値
 * 攻撃力、ダメージ双方に加算される模様
 * 突撃騎兵はこの値が特に高いので、一旦引いて何度も突撃を繰り返すことが重要
* 武器の長さ
 * 相手のリーチを上回っていると接敵されるまで攻撃を喰らわずに一方的に攻撃できる？
 * パイクには最長の5mが設定されており、パイク兵を特徴付けている
* ヒットポイント
 * 最終的に確定したダメージにより減らされいき、尽きるとロストする
 * 人体の基本HPと、ユニットごとのボーナスHPが加算されている
 * 騎乗していると馬や象のHPも加算される

#### 射撃について

* 射撃性能
 * ユニットの持つ射撃武器自体には具体的な射撃性能は設定されておらず、使用する投射物に射撃性能が設定されている（通常弾と火矢を切り替えたりするため）
 * 射撃武器毎にどの投射物を扱うことができるかと、デフォルトでセットされている投射物の設定がある
 * 本ビューアで表示している射撃性能はデフォルトの投射物のもの
* 射撃のリロードについて
 * 投射物ごとに装填時間(sec)が設定されていて、これに対してユニットの持つリロードスキル(%)により短縮される。実際の装填時間 = 装填時間 * (100-リロードスキル値)/100
 * ゲーム上で表示されている1分あたりの射撃回数は60を前項の値で割ったもの
 * 本ビューアでは1分あたりのダメージ値と全弾のダメージ量も計算して表示している
* precursor属性
 * 射撃武器に設定されている真偽値
 * ピルムなど、切り込み前に先行して投げる射程40mの短距離投射物に設定されている
 * これが真だと残弾があっても投射攻撃の選択ができず自動的に投射される？射撃レンジの表示も出ない
 * 本ビューアでは突撃前射撃という表示にした
 * 投槍を持つ近接歩兵はほとんどこれだが、イベリア剣士やトゥレオス槍兵などはこの属性が無く、普通の散兵と同様に射程80mで弾数も多い
 * ゲーム上ではユニットのカテゴリによってフィルタされているのか、近接歩兵だと射撃性能が表示されない

## 参考：DBスキーマ

----

* ユニット周りのデータ表示するだけならそんなに手間でもないだろうと思ったらいろいろ参照関係があって大変だった
* だいたいこんな風になってると把握したので、mod作りとかで参考になればと載せておきます

#### ユニット情報

* `main_units`
 * ユニットの基本情報を格納
 * Rome2では各ユニットが陸・海のユニットを持つので、実体は参照先の`land_units`, `naval_units`
 * ここでのプロパティとしては部隊人数、コストなどを格納
* `land_units`
 * 地上戦でのユニットデータ
 * 攻撃値、防御値、突撃値、追加ヒットポイント、士気、所有弾数などがここで設定されている
 * 肉体情報である`battle_entities`や、武器防具の`unit_armour_types`, `unit_shield_types`, `missile_weapons`, `melee_weapons`などへの参照を持つ
* `naval_units`
 * 海上戦のユニットデータ
 * 現状では本ビューアで取り扱ってない
 * 勢力ごとの違いもあまり無さそうだし海戦はオートで飛ばしてしまうのでちゃんと調べてない
* `battle_entities`
 * 人間や馬や動物の実体データ
 * 移動速度やヒットポイントの基本値などがここで設定されている
 * 騎兵は人体と馬のヒットポイント両方持つことになるので体力高い

#### ユニットの装備品

* `unit_armour_types`
 * 鎧のデータ
 * 装甲値と対射撃ボーナスが設定されている
* `unit_shield_types`
 * 盾のデータ
 * 装甲値と、防御スキル値、射撃ブロック率が設定されている
* `melee_weapons`
 * 白兵武器のデータ
 * AP/通常ダメージ値と、対（歩兵、騎馬、象）ボーナス、武器の長さが設定されている
* `missile_weapons`
 * 射撃武器のデータ
 * どの弾が装填されているかというだけ
* `projectiles`
 * 投射物のデータ
 * 白兵武器と同様なダメージ性能データと、その他物理情報がいろいろ設定されている


#### ファクション情報

* `factions`
 * ファクション情報
 * 所属文化と、軍隊グループの参照を持つ
* `start_pos_factions`
 * キャンペーンに使用されるファクション
* `campaigns`
 * キャンペーンの一覧

#### ユニットの雇用設定

* `units_to_groupings_military_permissions`
 * どの軍隊グループがどのユニットを雇えるかの設定
* `units_to_exclusive_faction_permissions`
 * ファクション固有で扱えるユニット設定
 * 主にルシタニ周りがちょこっとあるだけ
* `commander_unit_permissions`
 * 指揮官がどのユニットを扱えるかの設定


## 仕様


* Angular-fullstackでのMEAN構成
* 公式ツールのRome 2 Assembly kitから生データとスキーマ設定を参照してmongoDBに入れてRestAPIを作った
* リスト表示モジュールは現在開発まっただ中でunstableなAngular UI Gridを使ってるので、いろいろ不具合あるかも
